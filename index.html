C/C++的区别

C语言的特点：

  1. 作为一种面向过程的结构化语言，易于调试和维护

  2. 表现能力和处理能力极强，可以直接访问内存的物理地址

  3. C语言实现了对硬件的编程操作，也适用于应用软件的开发

  4. C语言还具有效率高，可移植性强等特点

C++语言的特点

  1. 在C语言的基础上进行扩充和完善，使C++语言继承了C语言的面向过程的特点，又成为了一种面向对象的程序设计语言

  2. 可以使用抽象的数据类型进行基于对象的编程

  3. 可以负担起以模板为特征的泛型化编程

多态

一.什么是多态？

  • 动态多态

  1. 对象多态: 能够以多种形态来表示它

    类型：

    • 向上转型：派生类 → 基类 （派生类对象用基类的对象来表示它）

      缺点：由于派生类Human的对象的内存空间比基类的Animal的内存空间大，赋值的过程会覆盖掉不属于基类Animal的内存空间，造成内存切片，所以不推荐使用，而是使用下面的方式

  • 向下转型：基类→派生类 （基类对象用派生类的对象来表示它）

    弊端：向下转型不能直接用对象进行赋值，因为Animal的内存空间<Human的内存空间，C++从语法上杜绝了这种情况；

    ​ 而用指针的形式来访问方法是非常骚的操作，但如果用指针去访问成员属性或者访问里面有访问成员Human的成员属性，会访问到错误的内存空间，结果是随机的，所以不推荐使用这种方法！

    类型转换的几种方式：

  • 隐式转换:编译器自动帮你转换
  • 强制转换:()
  • 静态转换: static_cast<>()
  • 动态强制转换:dynamic_cast<>()

    • 概念：只能用于支持方法多态类型的指针，如果转换成功，返回指针，转换失败，返回NULL

    • 应用场景：（1）向下转换 （2）跨类转换

    • 注意事项：

      • 不能转换this指针
      • 当用于转换引用时，转换失败会抛出异常，一般不推荐转换引用
      • 调试的时候一般配合typeid运算符来使用

  1. 方法多态

    概念：在基类的函数前加上 virtual 关键字，在派生类中重写该函数，通过对象多态中的向上转型得到基类的指针，然后调用虚函数，它将会根据对象的实际类型来调用相应的函数。如果对象类型是派生类，就调用派生类的函数；如果对象类型是基类，就调用基类的函数

    虚函数的使用的条件：

    • virtual只能写在类的内部声明或者定义，不能把virtual写在类的外部定义中
    • 调用类的对象是无法使用虚函数的，必须使用基类指针来实现虚函数的调用
    • 虚函数在派生类和基类中必须具有相同的参数列表
    • 虚函数在派生类和基类中返回值要求基本一致，但是当返回类型为类类型的指针和引用除外
    • 虚函不能是函数模板

    访问修饰符:

    • override后缀可以强制要求检查函数是否是重载（覆盖）
    • final后缀可以终止函数的重载

    多态存在的坑：

    • 在基类的构造函数和析构函数里调用虚函数，处于安全的考虑，都是静态绑定，调用的虚函数都是基类的虚函数
    • 调用虚函数的基类版本要使用 [基类名::] 限定符
    • 默认实参虚函数中不起作用，编译前就已经决定了默认实参的值了
    • 派生类的析构函数必须是虚析构函数才会被调用，才不会导致内存泄漏

  • 静态多态

内存分配

C 的内存基本上分为 4 部分：静态存储区、堆区、栈区以及常量区。他们的功能不同，对

他们使用方式也就不同

1.栈：由编译器自动分配释放;

2.堆：一般由程序员分配释放，若程序员不释放，程序结束时可能由 OS 回收；

3.全局区（静态区)：全局变量和静态变量的存储是放在一块的，初始化的全局变量和
4.静态变量在一块区域，未初始化的全局变量和未初始化的静态变量在相邻的另一块区域
（C++中已经不再这样划分），程序结束释放；

5.另外还有一个专门放常量的地方，程序结束释放：
(a)函数体中定义的变量通常是在栈上；
(b)用 malloc, calloc, realloc 等分配内存的函数分配得到的就是在堆上；
(c)在所有函数体外定义的是全局量；
(d)加了 static 修饰符后不管在哪里都存放在全局区（静态区）；
(e)在所有函数体外定义的 static 变量表示在该文件中有效，不能 extern 到别的文件用；
(f)在函数体内定义的 static 表示只在该函数体内有效；
(g)另外，函数中的”adgfdf”这样的字符串存放在常量区。

4.常-量存储区：这是一块比较特殊的存储区，他们里面存放的是常量，不允许修改（当

5.代码块（.text段）：存放代码，不允许修改，可以执行

虽然内存被人为的分成了很多块，但是CPU如何处理我们定义的段中的内容，是当作指令执行，还是当作数据访问，还是当作栈空间，不会因为我们人为的设计和安排，CPU就会按照这样的安排进行操作，而是完全按照程序中具体的汇编指令，以及汇编指令对CS:IP，SS：SP，DS等寄存器的设置来决定的；因此我们可以在对应的位置（函数外，函数内）和对应的语法（malloc，static等）来让编译器生成对应的汇编代码。

git指南

集中式（svn）

  • 对比：因为每次存的是差异，需要的硬盘空间会相对的小一点，可是回滚速度会很慢

  • 优点：代码存放在单一的服务器上， 便于项目的管理

  • 缺点：服务器宕机：员工写的代码等不到保障

    ​ 服务器炸了：整个项目的历史记录都会丢失

分布式（git）

  • 对比：git每次存的是项目的完整快照，需要的硬盘空间会相对的大一点（团队对代码做了极致的压缩 最终需要的实际空间比svn躲不了太多，可是Git的回滚速度极快）

  • 优点：完全的分布式

  • 缺点：学习起来比svn陡峭

  1. 原理图（创建模型）
    工作区域
    • 工作区：我们平时写代码的地方，此时git不会管理我们的代码
    • 暂存区：放数据对象的地方
    • 版本库：最终的代码实现提交到这里 .git目录就是版本库
    • 远程仓库
    git对象

    • 数据对象：一个文件代表一个数据对象

      • 结构：key:value(hash/value)
      • 作用：存储文件内容
      • 缺点：记住文件的每一个版本对应的hash值并不现实
      • 缺点：文件名并没有被保存，我们仅保存了文件的内容

    • 树对象

      • 作用：解决文件名保存的问题，允许我们将多个文件组织到一起
      • 结构：hash：指针
      • 缺点：不知道hash值对应的是哪一个版本
      • 缺点：不知道这个版本的一些基本信息

    • 提交对象: 包含一个树对象条目，代表着当前项目快照，从这树对象开始我们就能找到所有提交的数据对象，从而形成Git中的一个版本

      • 作用1：给树对象做一层封装（包含项目的基本信息）解决了树对象存在的问题
      • 作用2：提交对象前有一个活动指针HEAD，指向master分支，当有最新的提交对象时，HEAD会携带着master分支往前走

    文件状态
    • 未跟踪/已跟踪
      • 已暂存/已修改/已提交

  2. 常用命令

    • git操作最基本的流程

  • git高层命令（crud)

  • Git高级命令（分支）

  • Git团队协作

- 本地分支
- 本地跟踪分支

计算机基础

编程的本质是什么？编程的本质其实就是跟计算机沟通的过程，让计算机帮我们干活，所以要了解编程的本质就要去了解计算机

  1. 计算机主要元器件

    显示器，鼠标/键盘这些我们天天接触，很好理解。硬盘和内存的共通之处就是存储数据，而因为硬盘离CPU比较远，所以叫做外部存储器；内存离CPU比较近，所以叫做内部存储器；它们的内部结构是怎么样的呢？看内存模拟这张图，一个小格只能放电流的两种状态，一种是高电频（二进制的1），一种是低电频（二进制的0）。而一个小格是一个1bit，8个小格就是一个字节(B)，1KB就是1024个字节(B)。内存和硬盘虽然都是存储器，他们的主要区别在于读写速度，硬盘的读写速度慢，内存的读写速度快。内存断电之后会把里面的数据全部清空，而硬盘断电以后不会把数据清空，所以内存不需要考虑把数据真正的写在格子里面，由于少了这一过程，硬盘的读写速度远远比不上内存的读写速度，当然它们的读写速度还取决于它们各自的设计工艺等。CPU又称为中央处理器，它主要擅长加减乘除这类的运算，而不擅长小数运算，GPU擅长小数运算，也就是显卡。

  2. 传输过程

    500*200 计算机是怎么运算的呢？

    首先CPU通过控制总线告诉内存要读还是写，通过地址总线去告诉内存要读或者写的地址，然后内存把数据通过数据总线传输给CPU，而这里的总线就是我们的电线，只能传输0/1两种情况，有多少条数据总线决定了一次可以传输多大的数据，我们平常说的32位CPU可以认为有32条数据总线，换算成GB就是4GB,而编程的本质就是CPU跟内存的数据传输过程，我们定义了一个数字255，到内存里就变成了111111(2^8-1)。这是因为内存的设计工艺，而我们人类擅长的是十进制，这中间就产生了很多进制问题。
